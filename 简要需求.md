### 通用 AI Agent 框架实施方案书

#### 一、 项目背景与目标

为了解决大语言模型在处理复杂、长周期任务时面临的“上下文溢出”与“逻辑失忆”等瓶颈，本项目旨在构建一个高泛化性、企业级的通用 AI Agent 框架。通过将底层状态编排与上层业务逻辑彻底解耦，利用“可编译规格（soul.md） + 渐进式技能（skills） + 标准化工具（MCP） + 可观测运行时（LangGraph）”的组合模式，实现智能体的快速实例化与可靠的自动化执行。

#### 二、 核心架构设计

**1. 基础设施层 (LangGraph 运行时)**

* **状态机编排：** 底层使用 LangGraph 将智能体工作流建模为状态图（StateGraph）。通过定义贯穿生命周期的全局快照（State）、执行具体逻辑的节点（Nodes）以及决定控制流向的条件边（Edges），实现复杂逻辑的循环与自适应跳转 `[1]`。
* **执行与容错：** 提供企业级的持久化执行（Durable Execution）与事件流传输（Streaming）能力，支持容错恢复和用于调试的“时间旅行”机制 `[2]`。

**2. 记忆与认知系统 (双层持久化与心理学防衰减)**

* **工作记忆 (Short-term Memory)：** 利用 LangGraph 的 Checkpointer 机制管理线程级（Thread-scoped）状态，维护单次会话中的即时对话历史、执行快照与临时上下文 `[3]`。
* **长效记忆 (Long-term Store)：** 使用 LangGraph 的 Store 组件实现跨线程的记忆共享与语义检索 `[3]`。
* **认知对齐 (PsychMem)：** 引入基于心理学的记忆重组机制，将随时间自然衰减的情景记忆（如历史错误轨迹）与必须永久保留的语义记忆（如强类型的架构规范、代码底线）严格隔离，防止关键准则因多轮对话而被模型遗忘 `[4]`。

**3. 规格定义层 (soul.md)**

* **可编译身份规格：** 摒弃硬编码，使用 `soul.md` 作为智能体实例的唯一身份与行为控制中心。明确声明智能体的角色（Role）、核心目标（Goal）、语言风格设定以及不可逾越的边界护栏（Guardrails） `[4]`。
* **上下文视图编译：** `soul.md` 作为一个动态模板，负责指示系统如何提取长效记忆，以及如何在当前任务节点下挂载必需的技能目录，从而为模型提供最纯净的执行视图 `[4]`。

**4. 能力扩展层 (SKILL.md 技能包)**

* **标准化技能封装：** 将经验做法与执行脚本打包为 `SKILL.md` 目录格式，包含 YAML 元数据（用于路由）和 Markdown 正文（具体的执行步骤与强类型的输入输出契约） `[4]`。
* **渐进式披露 (Progressive Disclosure)：** 采用按需加载机制。智能体常驻提示词中仅包含极度轻量的技能元数据目录；只有当模型判定需要时，才将数百行的 `SKILL.md` 正文加载进当前工作记忆中，极大节省 Token 开销并避免无关技能带来的逻辑干扰 `[4]`。

**5. 工具接入网关 (MCP 协议)**

* **全局统一接入：** 通过集成模型上下文协议（Model Context Protocol, MCP），使用 `langchain-mcp-adapters` 库，为智能体无缝接入本地脚本、企业内部数据库及第三方 API 服务器 `[5]`。

#### 三、 关键运行工作流与治理机制

**1. 动态上下文发现与主动压缩 (Auto Compact)**

* 为了突破极限上下文窗口的约束，系统内置 Summarization（摘要）中间件与压缩节点。当可用 Token 水位逼近预设限制时，自动拦截常规流程，将漫长的交互历史提炼为高密度的交接摘要并重写回状态中，实现无损的滚动记忆 `[6]`。

**2. 多智能体交接与层级路由 (Handoffs)**

* 在面临高度复杂的复合型任务时，采用多智能体协作模式。通过工具调用（如 `transfer_to_agent`）更新共享的全局状态变量，显式触发条件边，实现控制权与上下文信息在不同“专家”智能体之间的顺滑转移 `[7]`。

**3. 运行时安全与拦截链**

* **人在回路 (HITL)：** 对于敏感的工具操作（如数据库写入或财务操作），通过运行时拦截器挂起（Interrupt）当前超级步，强制要求人类操作员审查并审批后方可恢复图状态执行 `[2]` `[5]`。
* **供应链沙箱：** 所有的外部 `SKILL.md` 脚本执行均必须被限制在 Docker 等隔离沙箱内，结合数字签名与权限白名单，阻断恶意技能包带来的系统性风险 `[4]`。

#### 四、 快速实例化实施路径

1. **基座部署：** 搭建 LangGraph 底层图执行引擎，并挂载 Redis 或 Postgres 作为 Checkpointer 与 Store。
2. **能力池建设：** 开发企业内部系统的 MCP 接口服务器，并建立规范的 `SKILL.md` 领域专家知识库。
3. **智能体编排：** 为不同的业务场景（如研发助手、自动化运维、客服流转）编写独立的 `soul.md` 身份配置文件。
4. **测试与上线：** 系统加载 `soul.md` 动态编译实例化 Agent，进行工作流试运行与回溯调试，最终投产至业务环境。
