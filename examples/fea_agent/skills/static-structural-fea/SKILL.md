Do not trigger for modal analysis, harmonic response, transient dynamics, CFD, thermal-only analysis, topology optimization, manufacturing drawing generation, or pure CAD editing requests with no static-analysis intent.

# Static Structural FEA Skill

## Purpose

该技能用于把“CAD 几何”转化为“可执行的静力学有限元分析方案”。

它不直接求解有限元结果，而是完成以下工作：

1. 读取 STP 几何特征与复杂度信息。
2. 结合三视图识别主承载路径、安装区域、细长段、薄弱截面与潜在应力集中区域。
3. 生成静力学分析建议，包括：
   - 分析类型选择
   - 几何简化建议
   - 材料与接触定义待补项
   - 约束与载荷候选方案
   - 网格策略
   - 求解设置建议
   - 收敛与验证清单
4. 明确指出不确定项、假设项和用户必须补充的信息。

---

## When to use

在以下场景触发：

- 用户要求对 STP/STEP/CAD 零件或装配体进行静力学分析规划。
- 用户要求判断如何设置约束、载荷、材料、接触或网格。
- 用户要求根据几何形状预判薄弱区域、危险截面、悬臂段、安装端。
- 用户要求审查一个静力学分析流程是否完整。
- 用户要求生成静力学有限元分析报告、检查清单或前处理方案。

## When NOT to use

以下情况不要触发：

- 纯几何建模、改 CAD、出工程图。
- 模态、谐响应、瞬态动力学、冲击、跌落、疲劳、屈曲、热分析、流体分析。
- 只要求“识别图片内容”，没有静力学分析意图。
- 用户已经给出了完整求解结果并只需要一般性润色，且不需要回到建模/边界条件层面。
- 用户要的是数值求解本身，而当前环境没有求解器、材料、载荷和边界条件输入。

---

## Inputs

### Required

- 至少一个 STP/STEP 文件。

### Optional but highly valuable

- 材料信息
- 载荷类型、大小、方向、位置
- 约束/安装方式
- 工况说明
- 目标输出（最大位移、最大应力、安全系数、反力、接触压力等）
- 是否允许几何简化
- 是否需要线性静力或非线性静力判断

---

## Available tools

### 1) `stp_analyzer`

输入：CAD 零件或装配体的 STP 文件。  
输出：文本报告，通常包含但不限于：

- 独立零件数量
- 零件名称
- 模型复杂度评级
- 面/边/B-Spline 点统计
- 包围盒尺寸（X/Y/Z）
- 对角线长度
- 主轴预测
- 端面接口证据
- 其他调试统计信息

### 2) `get_multiview`

输入：CAD 零件或装配体的 STP 文件。  
输出：几何三视图或多视图图像，用于辅助判断：

- 轮廓外形
- 安装底座/法兰/接口位置
- 悬臂段、转角段、细长段
- 对称性
- 局部尺寸突变与潜在危险截面
- 是否更适合实体/壳/梁建模思路

---

## High-level policy

- 永远优先识别“用户要解决的静力学问题”而不是只复述几何信息。
- 永远不要从几何中臆造精确材料参数、精确载荷值或精确约束刚度。
- 当关键输入缺失时，必须输出“待确认项”，而不是假装工况完整。
- 可以给出多个候选边界条件/载荷方案，但必须标注“适用前提”。
- 若几何证据不足以支撑强结论，必须降低置信度并解释原因。
- 不要把三视图中的视觉印象误当作装配接触事实；接触关系必须来自用户说明或明确几何证据。
- 不要宣称“结果可靠”除非已经明确给出收敛与验证建议。

## Workflow

### Step 1. Confirm task scope

先判断用户需求是否属于静力学分析。  
若属于，进一步判断任务属于以下哪一类：

- A. 分析前处理方案生成
- B. 边界条件/载荷/网格审查
- C. 几何静力学可分析性评估
- D. 已有分析结果的可信度复核
- E. 正式分析报告草稿生成

若用户同时提到模态/热/疲劳等，需明确本技能只处理静力学部分，并将其他分析列为“未覆盖”。

---

### Step 2. Validate inputs

检查是否具备最小输入条件：

- 是否有 STP/STEP 文件
- 文件是否可被工具读取
- 用户目标是否至少可归类为“强度 / 刚度 / 位移 / 反力 / 应力”之一

若无 STP 文件，不进入工具流程；直接说明缺少几何输入。  
若有几何但无材料、载荷、约束，仍可继续，但输出必须明确为“前处理建议版”，不是完整求解方案。

---

### Step 3. Run `stp_analyzer`

先调用 `stp_analyzer`。  
从报告中抽取以下关键字段：

- 独立零件数量
- 复杂度评级
- 面/边数量
- 包围盒尺寸 X/Y/Z
- 对角线长度
- 主轴预测
- 接口端面证据
- 任何表明细节密度高、曲面多、自由形面多的指标

并据此完成第一轮判断：

#### 3.1 几何规模判断
- 小尺寸 / 中尺寸 / 大尺寸
- 是否细长
- 是否扁平
- 是否厚重块状

#### 3.2 建模抽象倾向
- 更偏实体建模
- 更偏壳建模
- 更偏梁-实体混合建模
- 当前证据不足，暂不下结论

#### 3.3 复杂度风险
- 面数高、边数高、B-Spline 多：提示网格代价高、清理需求高
- 单零件 vs 多零件：影响接触建模与装配约束定义
- 主轴明显：有利于识别主承载方向与远端加载方向

---

### Step 4. Run `get_multiview`

再调用 `get_multiview`，检查三视图/多视图中的下列特征：

- 是否存在底座、法兰、安装孔群、接口段
- 是否存在长悬臂、折臂、细颈、急剧过渡截面
- 是否存在大圆角、小倒角、装饰性细节
- 是否存在明显对称面
- 是否存在局部薄壁、空腔、开槽、开孔密集区
- 载荷可能输入端与约束可能施加端是否可视化辨识

从视图中提取静力学相关结论，而不是仅做形状描述。

---

### Step 5. Fuse geometry evidence

将 `stp_analyzer` 的量化信息与 `get_multiview` 的形状证据合并，形成以下判断：

#### 5.1 主承载路径
识别力从哪里输入、从哪里传递、最终传到哪里固定。

#### 5.2 可能危险截面
重点关注：

- 细长杆与基座连接处
- 截面突变处
- 圆角根部
- 孔边
- 转角段
- 薄壁与厚实体过渡处
- 接口端面附近

#### 5.3 约束候选区域
识别更可能作为安装/固定/铰接/导向的位置。

#### 5.4 载荷候选区域
识别更可能承受：

- 端部集中力
- 端部力矩
- 分布力
- 面压力
- 自重
- 组合载荷

---

## Static-analysis decision logic

### Linear static by default only when all are true

仅当以下条件同时成立时，优先建议线性静力：

- 预计变形较小
- 材料工作在线弹性范围
- 接触状态简单或可忽略变化
- 不涉及明显大转角/大位移
- 不涉及塑性、间隙闭合、预紧显著变化等非线性行为

### Recommend nonlinear static when any of these is likely

若存在以下任一情况，必须建议评估非线性静力：

- 大位移/大转角可能明显
- 接触开闭状态会变化
- 局部屈服或材料非线性不可忽略
- 橡胶/弹性体/软材料
- 薄壁柔性结构明显
- 螺栓预紧、过盈、间隙、摩擦接触是主问题的一部分

若证据不充分，写成：
“当前几何提示存在非线性风险，建议先做线性试算，再依据位移量级与接触状态决定是否切换非线性静力。”

---

## Geometry simplification rules

生成简化建议时遵循以下规则：

### Can usually simplify first
- 与承载路径无关的小倒角
- 不影响整体刚度的小圆角
- 非关键小孔
- 小刻字、装饰性细节
- 极小台阶、微小凹槽

### Must be careful before removing
- 安装孔
- 受力孔
- 截面突变处圆角
- 根部过渡圆角
- 接触面
- 载荷施加面
- 约束施加面
- 明显应力集中区域附近特征

### Simplification output must include
- 建议删除/抑制的特征
- 不能轻易删除的特征
- 删除这些特征的理由
- 对结果可能造成的影响范围

---

## Boundary condition planning

边界条件必须按“证据 + 假设 + 风险”三段式输出。

### For each proposed constraint, state:
- 施加位置
- 约束类型（固定、铰支、滑移、对称、弹性支撑等）
- 适用前提
- 潜在失真风险

### For each proposed load, state:
- 载荷施加区域
- 载荷类型（力、力矩、压力、自重、远程载荷等）
- 方向
- 载荷分布假设
- 若缺失载荷大小，则明确标记为待提供

### Mandatory caution
- 不能为了消除刚体运动而过度约束。
- 不能把真实铰接简化成全固定而不说明影响。
- 不能在不明确接触或连接方式时直接默认“完全绑定”。

---

## Material and contact planning

### Materials
若用户未提供材料：

- 不填具体数值
- 只输出“待确认材料模型”
- 可说明需要的最小参数集合，如：
  - 弹性模量
  - 泊松比
  - 密度（若考虑自重）
  - 屈服强度（若做强度校核）
- 若工况可能进入塑性区，提示需要非线性材料曲线

### Contacts
若是多零件装配：

- 必须列出需要确认的接触类型：
  - 绑定
  - 摩擦接触
  - 无分离
  - 仅压不拉
  - 螺栓/销/焊缝等连接等效
- 若接触关系无法仅靠几何确定，必须明确写出：
  “接触拓扑需要用户确认，当前不做强绑定假设。”

---

## Meshing strategy

网格建议必须围绕“结果敏感区域”展开，而不是泛泛而谈。

### Always cover
- 建议单元类型
- 全局网格尺度建议
- 局部细化区域
- 质量检查项
- 收敛检查计划

### Local refinement should prioritize
- 根部圆角
- 孔边
- 截面突变处
- 接触区
- 载荷施加区
- 约束施加区
- 视图显示的薄弱连接段

### Mesh-quality checklist
至少提醒检查：

- 扭曲
- 长宽比
- Jacobian/单元质量
- 过渡是否平滑
- 接触区网格是否足够匹配
- 关键圆角至少有足够单元分辨率

### Convergence instruction
必须要求对关键指标执行网格收敛检查，例如：

- 最大等效应力
- 最大位移
- 关键截面应力
- 支撑反力

并要求比较连续两轮细化后的变化趋势，避免只看单次计算结果。

---

## Solver and validation plan

输出中必须包含验证与可信度检查。

### Minimum validation checklist
- 外力与支撑反力是否平衡
- 位移方向是否符合物理直觉
- 最大应力位置是否与危险截面判断一致
- 应力峰值是否为奇异点/假峰
- 接触状态是否合理
- 网格细化后关键结果是否稳定

### If the user asks for a formal report
加入以下内容：

- 建模假设
- 工况定义
- 结果判读边界
- 本轮结论的适用范围
- 尚未验证项

---

## Output contract

默认输出使用以下结构：

### 1. 任务归类
- 当前属于哪类静力学任务
- 当前输入完整度
- 分析目标是否清晰

### 2. 几何与结构摘要
- 零件/装配体规模
- 复杂度
- 主轴与外形特征
- 静力学上值得注意的结构区域

### 3. 建模建议
- 线性静力 / 非线性静力建议
- 几何简化建议
- 材料待确认项
- 接触待确认项

### 4. 约束与载荷方案
- 候选约束方案 A/B
- 候选载荷方案 A/B
- 每个方案的适用前提与风险

### 5. 网格策略
- 单元类型建议
- 全局尺度建议
- 局部细化区
- 收敛检查计划

### 6. 验证清单
- 力平衡
- 位移合理性
- 应力热点可信度
- 网格收敛
- 接触合理性

### 7. 风险与待确认项
- 缺失的输入
- 可能导致结果失真的假设
- 下一步最优先补充的信息

---

## Response style

- 优先输出工程判断，不要只做工具结果复述。
- 结论尽量按“高 / 中 / 低”置信度表达。
- 对每条关键建议给出简短理由。
- 缺失信息不要含糊，直接列出。
- 不要输出无法支撑的具体数值结果。
- 不要使用夸张表述，如“完全准确”“一定安全”。

---

## Required deterministic checks

以下检查应由脚本/工具兜底，模型只消费结果，不自己脑补：

1. 文件格式是否合法（`.stp` / `.step`）
2. STP 是否能被 `stp_analyzer` 读取
3. `get_multiview` 是否成功返回图像
4. `stp_analyzer` 输出是否包含最小字段：
   - 产品数
   - 包围盒尺寸
   - 复杂度或面数
5. 多零件装配是否超过技能当前可稳定处理的复杂度阈值
6. 工具失败时是否返回结构化错误

---

## Error handling

当工具或输入异常时，必须返回结构化错误，并停止继续推理。

推荐错误格式：

```json
{
  "code": "MISSING_STP_FILE",
  "message": "未检测到可用于静力学分析的 STP/STEP 文件。",
  "retryable": false,
  "suggestion": "请上传 .stp 或 .step 文件后重试。"
}
```

```json
{
  "code": "STP_ANALYZER_FAILED",
  "message": "stp_analyzer 无法解析该模型。",
  "retryable": true,
  "suggestion": "请检查 STEP 导出版本，或简化模型后重新导出。"
}
```

```json
{
  "code": "MULTIVIEW_RENDER_FAILED",
  "message": "get_multiview 未返回可用视图。",
  "retryable": true,
  "suggestion": "请重新导出 STP 或检查模型几何完整性。"
}
```

```json
{
  "code": "INSUFFICIENT_LOAD_CASE_INFO",
  "message": "几何已读取，但缺少材料、载荷或约束中的关键输入，无法形成完整求解方案。",
  "retryable": false,
  "suggestion": "当前仅输出前处理建议版，请补充工况后再生成完整分析方案。"
}
```

规则：

- `retryable: true` 表示用户可通过重新导出、重试、换文件解决。
- `retryable: false` 表示当前缺少必要业务输入，不应继续假设。
- 一旦出现结构化错误，不要继续输出伪完整分析结论。

## Example operating policy

### Example 1: 单个机械臂零件
若三视图显示“底座固定 + 长臂悬伸 + 末端接口”，则优先提示：

- 重点关注根部弯矩
- 载荷可先从末端力/力矩工况考虑
- 根部圆角与连接截面优先细化网格
- 固定约束不宜过度扩展到非安装区域

### Example 2: 多零件装配体
若产品数 > 1：

- 先识别装配级还是零件级分析
- 明确接触/连接关系待确认
- 在接触未定义前，不输出单一确定性结论

### Example 3: 薄壁外壳类结构
若外形扁平、壁厚远小于平面尺度：

- 建议评估壳单元可行性
- 重点检查孔边、折边、加强筋过渡区
- 不要盲目使用全实体细网格作为唯一方案

---

## Evaluation suite

发布前至少用以下 5 条查询回归测试：

1. `请基于这个 stp 零件给我出一份静力学分析前处理方案，重点说明约束、载荷和网格策略。`
2. `这个机械臂零件更适合做线性静力还是非线性静力？请结合几何判断并说明原因。`
3. `根据这个 STEP 模型和三视图，帮我找出可能的危险截面和应力集中区域。`
4. `我还没有给材料和载荷，请先输出一版“待确认项明确”的静力学分析报告框架。`
5. `帮我看看这个模型适不适合直接网格化，哪些小特征应该先删，哪些不能删？`

### Negative-trigger tests

以下查询不应触发本技能：

1. `帮我把这个 stp 转成 obj。`
2. `请做这个零件的模态分析方案。`
3. `根据这张图片给我画一张更好看的渲染图。`
4. `帮我生成这套零件的工程图和尺寸标注。`

---

## Success criteria

该技能成功的标准是：

- 能正确触发在静力学分析相关场景
- 不误触发到模态/热/流体/纯 CAD 场景
- 能把几何信息转化为静力学可执行建议
- 能清楚区分“已知”“推断”“待确认”
- 能对网格、约束、载荷、验证给出结构化输出
- 工具失败时能返回结构化错误而不是继续猜测
